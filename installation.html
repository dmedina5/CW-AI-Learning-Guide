📦 Installation Guide

Get both Claude Code and Gemini CLI set up on your machine. Both tools require Node.js 18+ as a prerequisite.

Prerequisites

  • Git for Windows - Required for Git Bash (download here)
  • Claude Pro/Max account or Console access - For authentication
💡 Windows Native Installation

Claude Code now has native Windows support - no WSL required! This provides faster file access and better Windows tool integration.

Installation (Windows)

Option 1: PowerShell (Easiest)

  1. Open PowerShell as Administrator (Press Win + X, select "Windows PowerShell (Admin)")
  2. Run the installation command:
irm https://claude.ai/install.ps1 | iex
⚠️ Execution Policy Error?

If you get an execution policy error, first run:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then retry the installation command.

Option 2: Command Prompt

  1. Open Command Prompt (Press Win + R, type cmd, press Enter)
  2. Run this command:
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

Option 3: WinGet

winget install Anthropic.ClaudeCode

Option 4: npm (Cross-platform)

# Install globally via npm (requires Node.js 18+)
npm install -g @anthropic-ai/claude-code

Verify Installation

After installation completes, close and reopen your terminal, then run:

claude --version

You should see the Claude Code version number.

Run Diagnostics

# Check that everything is configured correctly
claude doctor

Authentication

First Run Authentication

  1. Navigate to your project folder:
cd "C:\path\to\your\project"
  1. Start Claude Code:
claude
  1. Authenticate when prompted (you'll need a Claude Pro/Max account or Console access)

API Key Authentication (Alternative)

# Set environment variable in PowerShell
$env:ANTHROPIC_API_KEY = "sk-ant-your-key-here"

# Or set permanently for your user
[System.Environment]::SetEnvironmentVariable('ANTHROPIC_API_KEY', 'sk-ant-your-key-here', 'User')

Windows-Specific Issues

"claude command not found"

The installer should add Claude Code to your PATH automatically. If not:

  1. Press Win + Pause/Break (or Settings > System > About > Advanced system settings)
  2. Click "Environment Variables"
  3. Under "User variables", find or create PATH
  4. Add: %USERPROFILE%\.local\bin
  5. Click OK and restart your terminal

"Git Bash not found"

  1. Install Git for Windows
  2. Find your Git Bash installation (usually C:\Program Files\Git\bin\bash.exe)
  3. Set the environment variable in PowerShell:
[System.Environment]::SetEnvironmentVariable('CLAUDE_CODE_GIT_BASH_PATH', 'C:\Program Files\Git\bin\bash.exe', 'User')
  1. Restart your terminal

Authentication Issues

Reset credentials by deleting the auth file:

# PowerShell
Remove-Item "$env:USERPROFILE\.config\claude-code\auth.json"

# Then restart Claude Code
claude

Gemini CLI offers a generous free tier: 1 million tokens, 60 requests per minute, and 1000 requests per day.

Installation

Step 1: Install Gemini CLI

# Install globally via npm
npm install -g @google/gemini-cli

# Or via yarn
yarn global add @google/gemini-cli

Step 2: Verify Installation

# Check version
gemini --version

# View help
gemini --help

Authentication

Option A: OAuth Login (Recommended)

# Authenticate with Google
gemini auth login

# This opens a browser for Google OAuth
# Follow the prompts to authorize

Option B: API Key

# Set environment variable
export GOOGLE_API_KEY="your-api-key-here"

# Get API key from Google AI Studio:
# https://aistudio.google.com/apikey
🔑 API Key vs OAuth

OAuth is easier and uses your Google account. API Key is better for automation and scripts.

🛠 Troubleshooting

Common Issues

Node.js Version Too Old

# Check your version
node --version

# If below 18, update Node.js:
# Using nvm:
nvm install 18
nvm use 18

Permission Denied on Install

# On macOS/Linux, you may need sudo:
sudo npm install -g @anthropic-ai/claude-code

# Better solution - fix npm permissions:
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
# Add to PATH in ~/.bashrc or ~/.zshrc:
export PATH=~/.npm-global/bin:$PATH

Authentication Fails

  • Check your API key is correct
  • Ensure environment variable is set in current shell
  • Try logging out and back in: gemini auth logout && gemini auth login
